•	To design the checkout flow effectively, I first analyzed what information each section (Cart, Address, Payment, and Confirmation) contains and how users interact with it. This helped me organize the UI using usability principles so users can easily understand, navigate, and complete their purchase with confidence.
•	If users get stuck at any stage of the checkout process—such as Address, Payment, or a Declined Card error—we need to understand why the issue occurred, where in the flow it happened, and how to help users recover and continue their purchase successfully.
•	Users complete purchases when they trust the final price and believe the value is worth the cost. Users abandon carts when uncertainty, doubt, or unexpected charges outweigh the perceived value.
